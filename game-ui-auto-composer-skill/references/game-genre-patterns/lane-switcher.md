# Lane Switcher Pattern

## Typical properties
- 2 or more lanes
- character switching is the core interaction
- obstacle readability is critical
- center or lane band must remain clear

## Default page set
- Home
- Gameplay HUD
- Result

## UI priorities
- Home emphasizes game hook and start action
- Gameplay HUD minimizes obstruction
- Result highlights score, rewards, retry, next

## Safe-zone rule
Protect the entire lane field. Do not place large panels over switch trajectories.

## Common asset usage
- character frames -> gameplay + home showcase
- button bases -> home CTA / result actions
- badges/icons -> top utility or result stats
