# HTML CSS Prototype Guidance

## Purpose
Translate the structured page plan into a fast prototype mindset.

## Use when
- the developer wants Codex-ready structure
- the team needs a previewable layout
- engine-neutral planning needs a visual web proxy

## Output guidance
For each page provide:
- DOM-like region structure
- layout method suggestions
- stacking order
- relative widths/heights
- responsiveness notes
- class naming ideas
- lightweight animation ideas

## Rule
Treat HTML/CSS as a prototype or implementation bridge, not as the only truth.
