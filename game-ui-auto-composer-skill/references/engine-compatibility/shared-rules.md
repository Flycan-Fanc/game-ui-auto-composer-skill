# Shared Engine Compatibility Rules

## Core compatibility strategy
Keep outputs:
- asset-splittable
- constraint-based
- componentized
- state-aware

## Required fields
- asset id and usage
- component type
- parent-child hierarchy
- anchor
- relative size and position
- states
- motion trigger notes

## Avoid
- giant flattened compositions
- fixed-pixel-only layout
- missing state definitions
- vague motion instructions
