# Anchor and Sizing Rules

## Prefer anchor-based layout
Use:
- top-left
- top-center
- top-right
- bottom-center
- left-center
- right-center
- full-background
- relative-to-parent sizing

## Avoid
- absolute pixel-heavy layouts as the primary representation
- fragile hard-coded positions with no parent relationship

## Size guidance
Use semantic size labels when useful:
- small
- medium
- large
- hero

Also provide relative guidance:
- width as percent of parent
- min interaction size
- spacing scale
